# PLAN.md — urban-education-data Package Redesign

## Decisions (do not re-litigate these)

| Decision | Choice | Reason |
|---|---|---|
| Return type | `EducationDataResult` object | Pattern 3: `.to_df()` / `.to_dict()` / `.count` |
| Pagination | Auto-fetch all pages by default | Callers should never get silent partial data |
| Scope | Schools + School Districts + Colleges (IPEDS) | Full portal coverage |
| PyPI name | `urban-education-data` | Scoped to Urban Institute source |
| Import name | `educationdata` | Preserve existing import; flag below |
| Python floor | 3.11+ | Use modern type hints, `tomllib`, etc. |
| Pandas | Soft/lazy dependency | Import only inside `.to_df()`; not required for `.to_dict()` |
| Method prefix | Drop `get_` | Redundant on a class; `api.ccd_directory(...)` reads better |
| Segment params | Replace boolean flags with `by: list[str]` | Hides URL structure from callers |
| Project manager | `uv` | Fast, modern Python package/environment manager |

### Open naming question (flag for human before publishing)
The Urban Institute's official R package is also called `educationdata`. The Python import name `educationdata` may conflict with a future official Python release. Before publishing to PyPI, confirm whether to keep `educationdata` or use `urban_education_data` as the importable name.

## Component 1: `EducationDataResult` (`_result.py`)

```python
from __future__ import annotations
from dataclasses import dataclass, field

@dataclass
class EducationDataResult:
    count: int
    results: list[dict]

    def to_dict(self) -> list[dict]:
        return self.results

    def to_df(self):
        try:
            import pandas as pd
        except ImportError:
            raise ImportError("pandas is required for .to_df(). Install it with: pip install pandas")
        df = pd.DataFrame(self.results)
        df.attrs["count"] = self.count
        return df
```

Key points:
- `pandas` is NOT in the import at module level — only imported inside `to_df()`
- `df.attrs["count"]` preserves the total record count on the DataFrame
- No inheritance, no magic — plain dataclass

## Component 2: Pagination helper (`_pagination.py`)

The API response envelope is: `{"count": N, "next": "https://...", "previous": "...", "results": [...]}`

The `next` field is a full URL for the next page, or `null` when done.

```python
def fetch_all_pages(session, url: str) -> EducationDataResult:
    all_results = []
    count = 0
    while url:
        response = session.get(url)
        response.raise_for_status()
        data = response.json()
        if not count:
            count = data.get("count", 0)
        all_results.extend(data.get("results", []))
        url = data.get("next")  # None when last page
    return EducationDataResult(count=count, results=all_results)
```

Key points:
- `count` is taken from the FIRST page response (total records in the API)
- Follows `next` URLs until exhausted — no manual page arithmetic
- All methods call this instead of `session.get()` directly

## Component 3: `EducationDataAPI` (`_client.py`)

### URL construction pattern

```
BASE_URL = "https://educationdata.urban.org/api/v1/"

schools:          BASE_URL + "schools/{source}/{endpoint}/{year}/[by_segments]/?[filters]"
school-districts: BASE_URL + "school-districts/{source}/{endpoint}/{year}/[by_segments]/?[filters]"
colleges:         BASE_URL + "colleges/{source}/{endpoint}/{year}/[by_segments]/?[filters]"
```

### The `by` parameter

Every endpoint that previously had a `_segment` variant now accepts `by: list[str] = []`.

Valid `by` values per endpoint are documented in each method's docstring. The method constructs the URL path segments from `by` in the correct order (the API is order-sensitive: e.g., `disability/race/sex/` not `race/disability/sex/`).

```python
def _build_segment_path(self, by: list[str], valid_combos: list[tuple]) -> str:
    key = tuple(sorted(by))
    combo_map = {tuple(sorted(c)): "/".join(c) + "/" for c in valid_combos}
    if key not in combo_map:
        raise ValueError(f"Invalid segment combination: {by}. Valid: {[list(c) for c in valid_combos]}")
    return combo_map[key]
```

### Query parameter construction

```python
def _build_url(self, path: str, **kwargs) -> str:
    url = self.BASE_URL + path
    if kwargs:
        url += "?" + "&".join(f"{k}={v}" for k, v in kwargs.items())
    return url
```

## `pyproject.toml`

uv uses standard `pyproject.toml`. Initialize the project with `uv init` then configure as below.

```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "urban-education-data"
version = "0.1.0"
description = "Python client for the Urban Institute Education Data Portal API"
readme = "README.md"
requires-python = ">=3.11"
license = { text = "MIT" }
dependencies = [
    "requests>=2.28",
]

[project.optional-dependencies]
df = ["pandas>=2.0"]

[tool.uv]
dev-dependencies = [
    "pytest>=8.0",
    "pandas>=2.0",
]

[tool.hatch.build.targets.wheel]
packages = ["src/educationdata"]

[tool.pytest.ini_options]
log_cli = true
log_cli_level = "DEBUG"
```

Key points:
- `pandas` is optional for end users: `pip install urban-education-data[df]`
- `requests` is the only hard runtime dependency
- `hatchling` as build backend — zero-config, uv-compatible
- Move pytest config from `tests/pytest.ini` into `pyproject.toml` under `[tool.pytest.ini_options]` — one config file
- uv dev deps include pandas so tests can use `.to_df()`

## uv workflow

```bash
# Initial setup (run once)
uv init --no-workspace
uv add requests
uv add --optional df pandas
uv add --dev pytest pandas

# Day-to-day
uv sync              # install/update all deps from uv.lock
uv run pytest        # run tests inside the managed venv
uv run pytest tests/test_ccd_directory.py  # single file
uv run pytest tests/test_ccd_directory.py::test_filter_by_identifiers  # single test

# Build and publish
uv build             # produces dist/
uv publish           # upload to PyPI (requires PYPI_TOKEN env var or uv config)
```
